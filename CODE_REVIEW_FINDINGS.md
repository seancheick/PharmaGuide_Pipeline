# Pipeline Code Review — Full Findings

**Date:** 2026-07-05
**Scope:** Entire pipeline — `clean_dsld_data.py` / `enhanced_normalizer.py` → `enrich_supplements_v3.py` → `coverage_gate.py` → `score_supplements.py`, plus all support modules (units, dosage, RDA/UL, matching, blend detection, ledger), orchestration (`run_pipeline.py`, `batch_processor.py`, `preflight.py`), every gate/validator, all reference databases in `scripts/data/`, all configs, and `constants.py`.
**Method:** 14 parallel deep-review passes (one per subsystem, the two large files split by line range), each requiring evidence-backed, traced findings only. Highest-severity claims were then **independently re-verified by executing the real code against the real databases** — every headline finding below reproduced. Test suite baseline: **884 passed, 1 failed** (`test_missing_match_tokens_report_empty` — expects an artifact generated by an archived script; environmental, not logic), 9 skipped, 7 xpassed (stale `xfail` markers).

**Severity legend**
- **P0 / CRITICAL** — wrong verdicts, safety-inverting output, or silent data loss at scale
- **P1 / HIGH** — systematic score distortion in common cases
- **P2 / MEDIUM** — wrong results in edge cases, or guardrails that silently don't guard
- **P3 / LOW** — latent risks, dead code, telemetry/report bugs

Findings marked ✅ were reproduced by executing code; ⚙ were verified by full static trace of both sides of the contract.

## P0 — CRITICAL

### C1. ✅ Colorant substring check corrupts the identity of common actives at the CLEAN stage
`enhanced_normalizer.py:1917-1940` — the "explicit dye" block runs **first** in `_perform_ingredient_mapping`, before probiotic/vitamin/alias matching, and uses unconditional substring tests against `color_indicators.json`.
Reproduced: `Riboflavin` → `"natural colors"`, `Beta-Carotene` → `"natural colors"`, `Mixed Tocopherols` → `"natural colors"`, `Turmeric Root Extract` → `"natural colors"`, `Amaranth Sprout Powder` → `"artificial colors"`; also Lutein, Lycopene, Zeaxanthin, Astaxanthin, Saffron, Elderberry, Spirulina, Curcuminoids. Everything downstream (quality matching, dosing, scoring) sees a colorant instead of the nutrient.
**Fix:** run dye classification only after identity lookups fail; whole-token match; gate to color context (or exclude names resolvable via `ingredient_alias_lookup`).

### C2. ✅ Safety-DB names collapse to generic keys → benign ingredients flagged harmful/banned
`enhanced_normalizer.py:1800-1805, :978` index `preprocess_text(standard_name)`, which strips parentheticals/prefixes/suffixes. Reproduced lookup contents: `"chromium"` → **Chromium (VI)** (plain "Chromium" row flagged as a carcinogen), `"vitamins"` → Synthetic Vitamins, `"b vitamins"` → Synthetic B Vitamins, `"corn"` → Corn Oil (Refined), `"canola"` → Canola Oil (Refined), and in the banned fast-lookup `"green tea"` → *Green Tea Extract (High Dose)* — every green-tea product matches a >800 mg-EGCG-specific restriction. 44 safety entries collapse this way. Related (✅): `d-Alpha Tocopherol` (natural vitamin E) → **"Synthetic Vitamins"** because `d-`/`dl-` prefixes are stripped to the same key and the IQM key uses a hyphen (`alpha-tocopherol`) while the label yields a space.
**Fix:** for harmful/banned DBs, match only curated `match_rules.label_tokens`/aliases (never preprocessed `standard_name`); add a min-specificity guard rejecting single-generic-token keys; stop stripping `d-`/`dl-`/`natural`/`synthetic` from identity keys.

### C3. ✅ Negated "Contains no …" statements parsed as POSITIVE allergens, then laundered past enrichment's negation guards
`enhanced_normalizer.py:2680, 2688-2738` — the `contains:` branch has no negation check and runs on every statement type. Reproduced: statement `"Contains no sugar, salt, dairy, yeast, wheat, corn, soy or milk."` (type `Formulation re: Does NOT Contain`) → `parsed.allergens = ['wheat','soy','milk']` **plus synthesized warnings** `["Contains: Milk", "Contains: Soy", "Contains: Wheat"]` (while `allergenFree` simultaneously lists them). Enrichment re-parses the synthesized "Contains: Milk" string — negation already destroyed — and registers presence_type `contains`, the highest, un-negatable precedence. Also substring FP: `"Contains milk thistle…"` → milk allergen.
**Fix:** skip negated statement types/phrases before the `contains:` branch; require allergen-context keywords (`milk(?!\s+thistle)` or DB aliases); never synthesize positive "Contains: X" from text that didn't assert containment.

### C4. ✅ Allergen suppression trio in enrichment (false negatives — worst direction for an allergy app)
1. `_is_negated` is applied to **structured ingredient rows** (against its own docstring): `"No artificial colors. Contains milk protein isolate."` + ingredient `Milk Protein` → **milk allergen suppressed** (`\bno\s+` + `.*?` crosses sentences). `enrich_supplements_v3.py:4606, 4680-4692`.
2. Whole-statement negation: `"Made without gluten. Contains milk and soy."` → **zero allergens extracted** (`:4491-4503`).
3. Marketing claims override structured data: `parsed.allergens: ["milk"]` + targetGroup `"Dairy Free"` → allergen silently dropped instead of surfacing a conflict (`:4397-4443`).
**Fix:** never negation-filter ingredient-row allergens; per-clause negation scope; contradictions become conflicts, not suppressions.

### C5. ✅ Jurisdiction-blind banned matching → false US UNSAFE verdicts
`enrich_supplements_v3.py:4154-4171` exports only global `status`/`severity_level`; the DB's `jurisdictions[]`/`banned_countries` are never read. Verified DB: **Garcinia Cambogia** — `status: banned, severity: high`, jurisdictions **France (banned), Australia (restricted), no US entry** → B0 fires **UNSAFE** for a lawful, ubiquitous US retail ingredient. Same class: Kava (US = FDA advisory only). 95/115 matchable entries are critical/high.
**Fix:** derive `banned_in_us` (from `jurisdictions[region==US]`, falling back to `legal_status_enum ∈ {banned_federal, not_lawful_as_supplement, controlled_substance}`), pass jurisdictions through, gate UNSAFE on US applicability; correct DB rollups.

### C6. ✅ Nutrition extraction is last-write-wins with substring names → false "sodium free" etc.
`enhanced_normalizer.py:2994-3077` — reproduced: actives `Sodium 140 mg` + inactive `Sodium Benzoate` → `nutritionalInfo.sodium = {amount: 0.0}` → enrichment reports `sodium_free`, **`hypertension_friendly: true`** for a 140 mg-sodium product. Also `Sodium Hyaluronate 100 mg` → sodium=100 mg; `Calories from Fat 15` overwrites `calories 120 → 15`; `Sugar` (inactive, no qty) zeroes real sugars.
**Fix:** exact-name (or ingredientGroup) matching; first-match-wins; never overwrite with quantity-less rows.

### C7. ✅ Unit math cluster — 1000× dose errors and dropped nutrients
1. **No unit reconciliation anywhere:** `rda_ul_calculator.py:280-292` divides `amount / rda_ai` without comparing the passed unit to the DB unit; `unit_converter.py:193-207` returns failure (no mass fallback) for every nutrient without a vitamin rule — i.e. **all bulk minerals**. Reproduced: **Copper 2 mg → 0.22% RDA "deficient"** (2000 mcg → 222% "high"), `scoring_eligible=True`; Calcium 1 g → 0.1%; **Magnesium 0.4 g silently misses a real UL breach**. Same root: grams-based DB rows (Omega-3, Taurine 0.5 g, Creatine 2 g) get mg amounts → `Taurine 1000 mg` = **200,000% RDA**, "excessive", 0 points.
2. **Sugar/sodium unit detection by substring:** `'mg' in unit` / `'g' in unit and 'mg' not in unit` — spelled-out `"Milligram(s)"` → sodium ×1000 (55 mg → 55,000 mg, 2391% DV, false hypertension warning); `mcg` contains `g` → ×1000. `enrich_supplements_v3.py:7003-7005, 7050-7052`.
3. **`unit_aliases` table never loaded** (`unit_conversions.json:349-365` vs `unit_converter.py:131-145`): `ug`, `micrograms`, `grams`, `gm` all fail → nutrient dropped from adequacy/UL.
4. **Greek mu `μ` (U+03BC) unhandled** in `unit_converter.py` (only `µ` U+00B5) → `"5 μg"` B12 dropped. Same asymmetry in `normalization.normalize_text` (`:61-62`) → divergent normalized keys.
5. **Comma thousands:** RDA collector `float("1,000")` → `ValueError` → `continue` → nutrient silently dropped (`enrich:7336`; the other two parsers strip commas).
**Fix:** reconcile units against the DB unit before dividing (mass-convert or mark ineligible); explicit unit-alias map everywhere; load the alias table; `[µμ]g`; strip commas.

### C8. ✅ Orchestration data-loss cluster
1. **Resume double-skip** (`batch_processor.py:544-568`, confirmed by read): resume filters processed files **and** still skips `last_completed_batch+1` batches of the already-filtered list. 978 files @ batch 500: resume after batch 0 → `range(1,1)` → **zero files processed**, "Processing complete!", exit 0. Every resume silently drops up to `(k+1)*batch_size` files.
2. **Swallowed write failures** (`:899-931`): any exception in `_write_json_output` is logged and dropped; files were already marked processed → the batch is lost forever (resume skips it), summary still counts it "cleaned", exit 0.
3. **Stale outputs re-ingested** (multiple): no stage clears its output dir; deterministic `*_batch_N.json` names mean a smaller rerun leaves old high-numbered batches that enrich/score happily ingest → deleted/recalled products keep shipping.
4. **Clean stage exits 0 at 100% failure** (`clean_dsld_data.py:374-406`): `run()` returns True unconditionally; `run_pipeline` proceeds.
5. **Validator-crash products vanish** (`dsld_validator.py:99-101` + `batch_processor.py:851-859`): status `error` products are written to *no* output dir and no quarantine artifact.
**Fix:** reset `last_completed_batch` after filtering (or drop batch-index resume) + end-of-run count invariant; re-raise write failures and only mark processed after write; purge/manifest stage outputs; threshold-based exit codes; quarantine the `error` branch.

### C9. ✅ Manufacturer violations assigned to the WRONG company
`enrich_supplements_v3.py:6111-6146` + `_fuzzy_company_match:998-1009` — substring containment scores **1.0** with no length guard (rapidfuzz `partial_ratio` reproduces it). Reproduced: `Genepro Medical Foods` ↔ `Genepro` = (True, 1.0); `Guru Nanak Herbals` ↔ `Guru Inc.` = (True, 1.0) → another company's recall deduction, stacking to the −25 cap. Note the asymmetry: the trusted-manufacturer **bonus** requires exact match; the violation **penalty** trusts fuzzy.
**Fix:** exact/alias only (mirror the bonus), or ≥0.93 + token-set equality + min-length guard.

### C10. ✅ Proprietary-blend detector triple fault
1. **Dedup broken** (`proprietary_blend_detector.py:281` vs `:334-336`): adds `blend_name` to `seen_blends` but checks `matched_text`; the purpose-built `dedupe_key` property is never called. Reproduced: same "Digestive Enzyme Blend" in actives + otherIngredients → **2 detections, −24** instead of −12; downstream dedupes keyed on differing per-field values don't save it (costs B5 points and the D2 full-disclosure point).
2. **Comma amounts crash** (`:455`): `float("1,000")` → ValueError → caller's broad except discards **all** detector evidence for the product (line 422 strips commas; 455 doesn't).
3. **Unanchored generic terms over all text** (`:193-197` + DB): "Natural Tropical **Fruit Blend** Flavor" → blend, disclosure `none`, −10; DB lists **"Whey Protein Blend" under Stimulant Blends** (severity high) → whey products get `risk_category=stimulant`, `has_high_risk_blend=True`.
**Fix:** dedupe on `dedupe_key`; `replace(",","")`; word-boundary anchors + require blend-context (amount/sub-ingredients) for statement hits; clean the term list.

`enrich:5291-5296` (`claimed` includes `fda_registered`) + projection `:5875-5882` (fda tier unreachable) + scorer `:910` (same trap). Extremely common label phrase → systematic **b4b 4.0 instead of 2.0**.

### H2. ✅ Identity/matching distortions in the quality map path
- **Plain "Curcumin" maps to its worst form** (`turmeric powder`, score 7 vs proper ladder to 18) — form-exact tier beats parent-exact and the DB aliases "curcumin" on the turmeric-powder form.

A prior collision-resolution pass (see `migration_report.json`) has regressed.
- **Suffix/prefix stripping collapses identity keys** (`normalization.py:247-265`): reproduced `Lemon` → **Vitamin B9 (Folate)** (via stripped "Lemon Extract" folate-source alias); `Lemon Oil` (flavoring) → Folate; `Fish Oil` → `fish`.
- **Precedence spec violation**: `candidate_sort_key` (`enrich:3470-3479`) puts `tier` and `match_source` above `priority`; MATCHING_PRECEDENCE.md mandates priority first. Verified: (`Citrus Bioflavonoids` w/ std `Quercetin`) → P2 generic beats P0 compound.

- **preprocess/normalize divergence**: `preprocess_text` (index+queries) lacks β→beta, µ/μ→mcg, en-dash→hyphen, comma strip → `β-Carotene`, `Omega–3` unmapped.
- **other_ingredients aliases map to the raw alias text** instead of `standard_name` (1,522 keys resolve to themselves — canonicalization defeated). `enhanced_normalizer.py:1838-1854`.

### H3. ✅ Branded-ingredient cluster
Cleaner **replaces `name` with the branded token** (`enhanced_normalizer.py:3341`): `"Magnesium Bisglycinate Chelate (Albion)"` → `name="Albion"` (Zn and Mg rows become identical); enrichment then matches on the token, and **8 tokens (Shoden, Albion, TRAACS, Superba, OmegaVia, Synapsa, Lactium, Setria) have no quality-map alias** → premium branded ingredients scored as unmapped (9) unless `standardName` rescues. Fix: keep the full name (token stays in `branded_token_extracted`), retry matching with `raw_source_text`, add the 8 aliases.

### H4. ⚙ Scoring-engine defects
- **`scoring_config.json` is largely dead**: only 5 config paths are read; every point table, threshold, grade scale, violation cap, POOR cutoff (<32) is hardcoded. Config edits are silent no-ops (values agree *today*; drift trap).
- **`shadow_mode: true` is read by nothing** — no shadow behavior exists; all verdicts ship.
- **Zero-scorable bypass**: `total_active` counts skipped rows; a blend-header-only product gets coverage 1.0, passes the mapping gate, scores ~35-40/80 **SAFE** instead of NOT_SCORED (`_mapping_gate:248-272`).
- **B0 order dependence** (confirmed by read, `:360-370`): first hard-fail wins — critical/high before recalled → UNSAFE (score 0.0) instead of BLOCKED (score null).
- **Multivitamin A1 "smoothing" is a blend, not a floor**: premium multi (avg 18) → A1 11.05 vs 13.0 if not classified multivitamin (`:430-433`); SCORING_LOGIC describes a floor.
- **Regression guard can be neutralized** when a banned exact/alias hit maps to a recognized-non-scorable ingredient (`min(unmapped_count_raw, …)` sizing, `:252-256`); exposure limited to low/missing severity.
- SCORING_LOGIC.md is a stale v3.4.0 spec (A30/B45/C15/D8, floor 10) — do not "fix" toward it.

### H5. ✅ Serving/dose basis distortions
- **Multi-serving mismatch**: cleaner picks canonical quantity = `variants[0]` (often the child serving) and reads serving context from the wrong dict (never emitted); enrichment picks the **max** serving for `serving_basis` → child 10 mg paired with adult 2-gummy basis = adequacy at half dose; `quantityVariants` has zero consumers. `enhanced_normalizer.py:4328-4375` vs `enrich:6902-6919`.
- **Directions parsing**: "units per intake" conflated with servings/day, frequency multipliers ignored ("2 capsules twice daily" → 2), precaution text ("Do not exceed 6 tablets in 24 hours") parsed as dosage → up to 6× per-day inflation and false over-UL (`enrich:6798-6802, 6927-6988`); `_extract_directions` truncates at the first newline, losing "Do not exceed" ceilings (`normalizer:4650-4653`).
- **Adequacy at MAX servings vs dosage_normalization at MIN** in the same record → contradictory per-day numbers; "1-3 caps" vitamin C = 1667% "excessive"/0 pts vs 556% "high"/2 pts (`enrich:7385-7391` vs `dosage_normalizer:478`).
- **RDA/UL computed for Male 19-30 only** and `data_by_group=[]` stripped → Iron 18 mg = 225% "high"/2 pts (male RDA 8) vs female 100% "optimal"/3 pts; pregnancy ULs never applied; device can't recompute (`enrich:7387-7391, 7460`; `rda_ul_calculator.py:140-147`).
- **No-UL nutrients banded "excessive"** purely on %RDA>300 (B12, biotin, K) (`rda_ul_calculator.py:37-43, 491-493`).
- **Folate UL basis**: converter ×1.7 → DFE compared against UL 1000 "mcg DFE", but the IOM UL is 1000 mcg **folic acid** (=1700 DFE) → over-UL fires at ~588 mcg folic acid; unknown-form folate defaults to folic acid ×1.7.

### H6. ✅ Crash-to-product-loss paths
- `percent_share=None` → `TypeError` in `_match_multi_form` sum → **whole product `enrichment_status=failed`** ("Vitamin A (as 100% beta-carotene and retinyl palmitate)"); root cause: dead `>1.0` redistribution branch (`enrich:2687, 2762, 851-873`).

- Unanchored `ENHANCED_EXCLUSION_PATTERNS` (`re.search`): active `"Green Tea Extract (less than 2% caffeine)" 500 mg` → treated as nutrition fact → **ingredient vanishes** (`normalizer:4961-4964`).

### H7. ✅ Cross-text false positives inflating scores
- **Standardization % bleeds across ingredients**: first "standardized to X%" anywhere credits every botanical (plain Ginkgo credited 98% from Green Tea's text) → A5b (`enrich:3800, 3846-3848`).
- **Synergy**: one ingredient matches multiple cluster variants (Magnesium alone qualifies 11 clusters via `magnesium` + `magnesium glycinate`) → A5c for single-ingredient products; dose check unit-blind (Melatonin 300 **mcg** passes a 0.5 **mg** floor) (`:3916-3949`).
- **Delivery substring over all text**: "micellar casein", "hydrolyzed collagen", "fermented barley" → tier-1/2 delivery-technology credit (A3) (`:3622-3625`).
- **BRAND study guard defeated**: `_brand_mentioned` checks the same alias list that includes generic names → generic "Astaxanthin" credited the AstaReal RCT (Section C); 5/57 brand studies confirmed exploitable (`:5942-5978`).
- **Probiotic strain double-count**: nested rows are emitted twice by the cleaner (hoisted + embedded) → `total_strain_count` doubles → CFU/diversity tiers inflate (`normalizer:3244-3266` + `enrich:6284-6292`).
- **CFU**: scientific notation ("5 x 10^9 CFU") unparsed → 0 CFU points; statement-level totals counted once per probiotic row (3 rows × "50 billion" statement = 150B); "trusted by 2 billion people" parses as CFU (`enrich:6276-6291, 6441-6469, patterns :718-720`).

### H8. ✅ Misclassification of scorable rows
- `EXCIPIENT_NEVER_PROMOTE` bare substrings: **"watercress"⊃"water"**, "honeysuckle"⊃"honey" → real actives skipped as `recognized_non_scorable` with `identity_confidence=1.0` (`enrich:1863-1865, 2135-2137, 2448-2456`).
- Potency regex `\d+\s*(mg|mcg|g|μg|iu)` without trailing boundary: "omega-**3 g**old", "d**3 g**ummy" count as high-signal potency → junk rows scored (`:2460-2482`).
- `hierarchyType` is a dict but compared to strings → header detection leg dead (`:2066-2067`); the `'blend' not in name.split()[-1:]` carve-out exempts names ending in "blend" from structural-blend evidence (`:2061-2065`) → dosed "Energy Blend 500 mg" rows scored as single actives.
- Multi-form renormalizes over matched forms only: 80%-unmatched form → 100% premium credit (`:2762-2770`).
- Promoted dose-less absorption enhancers (BioPerine) destroy the D2 full-disclosure point (`:2115-2122` + scorer `:1252-1260`).
- Quantity-less single ingredients declared `proprietaryBlend: true` with `transparencyScore: 100/"full"` + `individualAmounts: "Provided"` (vacuous `all()` on empty) (`normalizer:3234, 2876-2885, 4613-4616`).
- Clean-claims regex `No\s+…` without `\b`: "Contains Ami**no** Acids, Soy Lecithin" → **"Soy Free"** claim; whey+milk product → "Dairy Free" — and these feed `negated_terms`, which can suppress true allergens (`normalizer:4723`).

### H9. ⚙ CBD & friends dual-classified
`cbd` resolves to quality entry `cbd_full_spectrum` AND `BANNED_CBD_US` (`not_lawful_as_supplement`); also 7-keto DHEA, green tea, chromium, cupric sulfate. ## P2 — MEDIUM (guardrails that don't guard + edge-case wrongness)

### G1. ⚙ The only wired gate (coverage) doesn't check what it claims
- Correctness checks read fields that **don't exist** (`claims_data`, `allergen_data`, `rda_ul_data.analyzed_ingredients[].conversion_evidence`) — they exist only in tests, which fabricate them (`coverage_gate.py:504-511, 562-566, 590`). Note `collect_rda_ul_data` is **false** in `enrichment_config.json`.
- Absolute-count downgrade: products with <10 entities and ≤2 unmatched are WARN-only → for typical supplements the "99.5% BLOCK threshold" is really "allow 2 unmatched per product" (reproduced: 60% coverage passes); batches <50 products downgrade every BLOCK→WARN; `run_pipeline` can't set strict mode.
- Missing `match_ledger` → vacuous 100% coverage, `can_score=True`; `ImportError` on the gate module → warning + PASS (`run_pipeline.py:331-333`); unknown ledger domains never checked (fail-closed helper is dead code).

### G2. ⚙ Unwired/inert validators
- `enrichment_contract_validator.py` is **never invoked by the pipeline** (docs/config claim it enforces); `strict_mode` stored, never read; Rules B/E.4 read stale schema paths (`dietary_sensitivity_data.allergens`, `product.organic`) so they can't fire even when run.
- `identity_chain_verifier.py`: hardcoded output paths + 10 hardcoded IDs; with no data loadable prints "All 0 products passed", exit 0; latent `TypeError` (list fields used in int arithmetic).
- `score_stability_gates.py`: reads `caution_triggers`/`immediate_fail` the scorer never emits (2 of 4 gates permanently no-op); `rate_applicable` used before assignment → UnboundLocalError the moment triggers appear; `--expected-change` suppresses ALL failures if the file merely exists (content never read).
- `regression_snapshot.py`: missing/corrupt snapshot files become PASS (errors recorded as deltas, never alerts — reproduced); aggregate-only comparison (offsetting ±20 shifts pass); contradiction metric permanently 0 (stale paths); manifest hashes embed timestamps (never reproducible).

- `validate_json_files.py`: hardcoded `/Users/seancheick/...` log path (crashes on import elsewhere), stale schemas, always exit 0 — superseded by `validate_database.py`; delete.
- `pipeline_audit.py` / `claims_audit.py`: stale allergen path; sample the same glob-head every run; always exit 0.
- `preflight.py`: `configs_ok`/`scripts_ok`/`schema_ok` computed but excluded from the exit code; `--quick` ignores JSON corruption of critical files.

- Evidence-system scope validation truncates dotted paths (`labelText.raw` → `labelText` not in approved list) → all raw-label claims `scope_violation` (legacy paths still feed the scorer, so impact deferred).

## Cross-cutting root causes

1. **Substring matching without word boundaries** — appears in ≥10 findings across all stages (colors, excipients, salt/red-flag terms, delivery, sodium units, company names, hierarchy, claims).
2. **Asymmetric normalization** — at least 3 unit-normalizers and 3 text-normalizers that disagree (µ/μ, β, dashes, commas, hyphens); index and query sides normalized differently.
3. **Fail-open error handling** — broad `except` + log + continue converts crashes into silent data loss or missing safety signals (blend detector, RDA collector, batch writes, gates).
4. **Producer/consumer schema drift with no contract test** — validators and gates read fields that were renamed or never existed; tests fabricate the stale shapes instead of using real enrichment output.
5. **Config theater** — `scoring_config.json` point tables, `shadow_mode`, `enrichment_config` tuning keys, and `strict_mode` flags are read by nothing.
