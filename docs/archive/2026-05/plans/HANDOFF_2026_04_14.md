### Context

### Sprint 23a: CAERS Adverse Event Integration (HIGH PRIORITY)

**What:** FDA CAERS (Center for Food Safety adverse events) — real-world reports of people getting hurt from supplements. 8.5 MB bulk download at `https://api.fda.gov/download.json` under `/food/event`.

**Why:** This is the ONLY real-world pharmacovigilance signal in the entire pipeline. Currently we score ingredient quality (IQM) and check for interactions, but we have ZERO data on whether ingredients actually caused harm in practice. CAERS fills this gap.

**What CAERS is NOT:**
- NOT the same as harmful_additives.json (that's formulation quality — excipients like artificial colors, preservatives)
- NOT the same as banned_recalled_ingredients.json (that's regulatory actions — FDA bans/recalls)
- CAERS = "this active ingredient has a real-world adverse event track record"

**What IQM is (clarification for the next agent):**
- IQM scores active ingredients on QUALITY (bioavailability, form, absorption) on a 0-18 scale
- Low score = poor bioavailability/inferior form. High score = premium form, well absorbed
- IQM is NOT a bonus file — it's a quality assessment. A low IQM score means the ingredient is poorly formulated, not that it's harmful
- CAERS would add a SEPARATE safety dimension: "this ingredient may be well-formulated (high IQM) but has real-world adverse event reports"

**Implementation plan:**

1. **Download CAERS bulk data**
   - URL: `https://api.fda.gov/download.json` → find `/food/event` download links
   - Format: zipped JSON files, same schema as API
   - Store at: `scripts/data/fda_caers/` (raw downloads, gitignored)

2. **Build `scripts/api_audit/ingest_caers.py`**
   - Parse all CAERS reports
   - Filter to dietary supplement products (CAERS includes all foods — need to filter `products[].industry_code` or `products[].industry_name` for "Dietary Supplements" / "Vit/Min/Prot/Unconv Diet")
   - For each suspect ingredient, aggregate: total reports, serious outcomes (hospitalization, death, ER, life-threatening), outcome distribution
   - Match ingredient names to IQM canonical_ids via fuzzy matching + CUI lookup
   - Output: `scripts/data/caers_adverse_event_signals.json`

3. **Design `caers_adverse_event_signals.json` schema**
   ```json
   {
     "_metadata": { "schema_version": "1.0.0", "source": "FDA CAERS bulk download", "last_updated": "...", "total_ingredients": "..." },
     "signals": {
       "green_tea_extract": {
         "canonical_id": "green_tea_extract",
         "total_reports": 247,
         "serious_reports": 142,
         "outcomes": { "hospitalization": 89, "er_visit": 41, "life_threatening": 8, "death": 4 },
         "top_reactions": ["liver injury", "hepatitis", "jaundice", "abdominal pain"],
         "signal_strength": "strong",
         "notes": "Primarily associated with high-dose concentrated extracts (>800mg EGCG/day)"
       }
     }
   }
   ```

4. **Integrate into scoring** — New sub-component in Section B (Safety & Purity):
   - Strong signal (100+ serious reports): -2 points
   - Moderate signal (25-99 serious reports): -1 point  
   - Weak signal (10-24 serious reports): -0.5 points (flag only, informational)
   - Below 10: no penalty (noise threshold)
   - Add to `scoring_config.json` as `B_caers_adverse_events`

5. **Integrate into Flutter** — Add to product detail warnings:
   - "FDA has received X adverse event reports for [ingredient], including Y hospitalizations"
   - Source link to FDA CAERS

## KEY DECISIONS MADE

1. **Context-aware scoring:** Active-source harmful penalties suppressed for low/moderate severity. High/critical still fires. Based on research of Labdoor, EWG, ConsumerLab, Yuka — none dual-score.
2. **IQM alias additions are safe to batch** — additive-only, no scoring impact. New entries need full schema validation + UMLS verification.
3. **PMID content verification is non-negotiable** — run `verify_all_citations_content.py` after any citation change.
4. **Drug class gaps block interaction categories** — always check taxonomy before adding rules.
5. **CAERS is a separate data source from harmful_additives** — harmful_additives = formulation quality (excipients), CAERS = ingredient safety track record (adverse events on actives).
6. **IQM is NOT a bonus file** — it scores active ingredient QUALITY (0-18 scale: bioavailability, form, absorption). Low score = poor formulation, not harmful. High score = premium form. CAERS adds a separate SAFETY dimension.
